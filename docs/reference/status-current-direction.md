- **Stream-based coordination (append-log read by cursor)**: implemented per the stream-coordination spec. Coordination moves from turn-start room polling to append-only event streams read by cursor, passive by default, with an intentional-ping escalation and per-actor selective attention; CRDT stays on the graph (it converges shared STATE, not awareness). The named primitive is `rustyred-thg-core/src/stream.rs` (`StreamUrgency`/`StreamEvent`/`StreamLog`/`StreamRegistry`/`StreamKey`/`StreamDelta`, re-exported from the crate root): the append-only special case of `ordered.rs` -- keyed by a strictly-increasing `u64` ordering token that never moves, so `read_after(cursor, limit)` is an early-stopping `OrdMap` range over `(cursor, ..]` (O(result + log n), nothing in the window missed or duplicated, no merge step). `StreamLog::append` assigns the next token; `ingest` re-inserts an event at its own token (durable rehydration) while keeping `next_token` monotonic; `StreamRegistry` is the local-embedded live-tail transport bundling logs + per-`(actor, stream)` cursors + per-actor subscriptions. The MCP tool surface (`rustyred-thg-mcp/src/lib.rs`) is the durable delta-pull transport and replaces the room-poll surface: `stream_publish` (write), `stream_read` (read; passive-by-default, `advance=true` consumes the window once), `stream_subscribe`/`stream_unsubscribe` (writes). Each event persists as a `CoordinationStreamEvent` graph node (+ `COORDINATION_STREAM_EVENT_OF` edge to a per-`(tenant,topic)` `CoordinationStream` head node that holds the monotonic `next_token` allocator, reserved read-modify-write so concurrent publishers get distinct tokens); cursors are `CoordinationStreamCursor` nodes, subscriptions a per-actor `CoordinationStreamSubscription` node. The durable read rehydrates the matching event nodes into a transient core `StreamLog` and runs the identical `read_after` -- one model, two transports. The **intentional-ping bridge**: an `ask`/`block` publish with a `target_actor` writes the stream event AND calls the existing `write_coordination_message` with `delivery=wake` + the target as a mention, so a warm head drains it at its next Stop-hook mention pass and a cold head hits the existing wake -- the ping reaches the target even if it is not subscribed to the stream (mentions are actor-targeted, not subscription-gated).
