# App reference

Standalone crates and clients under `apps/`. Each depends on the `rustyredcore_THG` workspace but builds on its own.

| App | What it is |
|-----|------------|
| `browser` | `theorem-browser`: the Servo-embedded substrate-native browser. Standalone crate, not in the workspace. CI-only build. |
| `browser-substrate` | `theorem-browser-substrate`: the Servo-free page-to-substrate seam. Ingests a `LoadedPage` into a `GraphStore`. Builds in seconds. |
| `commonplace-api` | CommonPlace interoperability API seam. Serves the typed consumer GraphQL profile and MCP stdio tools over the `commonplace` object model, with optional durable RedCore plus disk backing; also exposes an embeddable loopback server for the desktop shell. |
| `commonplace-desktop-runtime` | Local CommonPlace runtime sidecar. Watches a canonical git tree, ingests changes into a durable RedCore-backed CommonPlace store, serves loopback control/data routes with pairing and local access, tracks run and presence state, and keeps sidecar writes out of the watched tree. |
| `commonplace-clipper` | Browser extension fork of `obsidianmd/obsidian-clipper` for CommonPlace push capture. Preserves upstream extraction, reader, highlighter, templates, and interpreter, and adds a CommonPlace save target that posts `CapturedObject` JSON with `captureMethod: "clipped"` to the configured endpoint. |
| `copresence-editor` | Browser adapter for `theorem-copresence`: Velt/Yjs plus Tiptap collaboration with an OpenAI-compatible Gemma co-writer seam. |
| `desktop` | Tauri desktop shell. Owns the local harness node, the CommonPlace API loopback server, keychain, receiver, browser tabs, and native command bridge; SPEC-9 repoints the main window to the CommonPlace Next.js frontend export. |
| `harness-console` | Standalone Next.js 16 / React 19 control surface for Theorems Harness at `harness.theoremsweb.com`. Greenfield app with Agent, Memory, Skills, Rooms, Runs, API Keys, Providers, Usage, Connections/MCP Hub, Settings, collaborative CodeMirror/Yjs editor, cosmos.gl memory graph, Dynamic Island omnibar, tokenized 4px design-math lint, and the CommonPlace code-workspace shell (`/api/theorem/agent`, `/api/commonplace/rustyred`, OpenUI/block-view registry, RustyRed data contract, deck.gl preview). |
| `ios` | `TheoremKit`: a Swift Package shared kit layer, distinct from `theorem-ios`. |
| `jobintel` | Standalone job-intelligence CLI. A light HTTP consumer of a running RustyRed; no path-deps into the substrate. |
| `notebook` | Python mirror of Theseus's inference layer: reference engines, the native-vs-Python routing kernel, byte-parity and cost gates. |
| `obsidian-sync` | `theorem-harness-sync`: device-side Obsidian community plugin (TypeScript). Mirrors memory docs into a vault and writes note edits back into the graph. |
| `orchestrate` | Python MAP-Elites orchestration tick (`map_elites_tick.py`). |
| `rustyred-embedded` | Embedded RustyRed engine over a local data directory. Exposes GraphQL/MCP in-process plus a durable DocTree filesystem (`fs_write`, `fs_read`, `fs_ls`, `fs_unlink`, `fs_rename`, `fs_mkdir`, `fs_rmdir`, `fs_is_dir`, `list_directories`), the W0 batch workspace seam, an opt-in post-File-write hook seam for domain maintenance, W4 shared code-embedding configuration for File vectors, and explicit `reembed_files` refresh for existing workspaces. |
| `rustyred-fuse` | Standalone W6 DocTree filesystem proof. Default builds and tests are pure Rust with no macFUSE/libfuse dependency; the optional `mount` feature enables `fuser` kernel-mount glue only for hosts that deliberately install the platform FUSE driver. |
| `rustyred-git` | Git-as-truth layer for code workspaces. Provides the full local VCS slice through a git-CLI backend, a tested pure-Rust `gix` backend slice for init/open/head/current-branch/read-at-HEAD/clone plus materialized-tree commit-all, `stage_all_from_worktree`, force checkout/rematerialization, clean merge/conflict surfacing, branch/worktree diff, `GixWorktreeStatus` commit-decision receipts, `GixIndexStatus` staged/unstaged/untracked porcelain, branch snapshot materialization for isolated parallel-agent work, push-ready ref update/object closure/packfile/receive-pack request generation, local bare push by consuming that closure plus fast-forward branch ref update, local receive-pack transport, GitHub-compatible Basic-auth smart-HTTP receive-pack push, and GitHub REST PR-open seam with mock plus live draft-PR proof. |
| `rustyred-workspace` | Code-workspace import and execution seam. Imports a materialized git checkout into `rustyred-embedded` with gitignore-aware source filtering and one batch DocTree persist; URL imports reuse CodeCrawler's credentialed staging path. Also installs the W1 direct File-write CodeCrawler hook, materializes DocTree files to a real OS directory, runs local toolchain commands with env stripping/timeout, syncs changed source files back while excluding build artifacts, optionally feeds changed source bytes into W1 CodeCrawler indexing, drives the same source-rewrite loop through the receiver `SandboxRuntime` put/run/get contract via `LocalProcessSandbox`, exposes a streaming sandbox bridge that can cancel from stdout while still syncing source edits made before cancellation, provides the W6 `DocTreeMount` core for read/list/getattr/write/mkdir/unlink/rmdir/rename with artifact throwaway routing, directory prefix rename, persistent empty directory markers, and rmdir outcome mapping, and has an opt-in `fuse-host` feature for a compile-tested `fuser` host adapter over a thread-safe backend seam plus `EngineFuseBackend`, an actor-backed real-engine implementation of that seam, configurable POSIX attrs, recursive inode cleanup, temp-backed artifact storage, source/artifact rename crossings including directory trees, and write-buffer cleanup on inode forget. |
| `rustyred-proxy` | First-class local Anthropic Messages proxy binary over the `theorem-agentd` proxy module. Runs the proxy-only local node path for `/v1/messages`, native tool-output membrane sampling, ambient context injection, and PR #60-style local co-presence via `/v1/presence*`. |
| `theorem-agentd` | Local assistant daemon runtime: an OpenAI-compatible model loop, local GGUF models, MCP tool host config, and the receiver sidecar. Ships both `theorem-localmodel` (descriptive daemon alias) and `theorem-agentd` (compatibility binary). |
| `theorem-grpc` | Theorem's first gRPC server. Serves `theseus_search.v1.SearchService` in pure Rust over the substrate. Own Railway service. Also exposes `/health` liveness, `/ready` RedCore readiness, and Theorems Harness remote-doctor probes for queue, dependency, and tenant guardrail diagnostics on the shared listener. |
| `theorem-harness-node` | Node.js (NAPI-RS) binding over the `theorem-harness` Rust SDK. A thin shell so the Node surface cannot drift from the core. |
| `theorem-harness-server` | Standalone Axum JSON/HTTP transport over `theorem-harness-runtime`. Serves run list and detail plus coordination reads for the iOS and web surfaces. |
| `theorem-harness-swift` | Swift (UniFFI) binding over the `theorem-harness` Rust SDK. Same surface as the Node binding, generated from the same core. Serves the iOS app. |
| `theorem-ios` | SwiftPM native iOS client scaffold: SwiftUI shell, Dynamic Island control surface, projection picker, smoke executable. |
| `web` | Empty local web staging directory seen by full doc-drift checks. Not a tracked product surface yet; keep product web work in `harness-console` or `theorem-gateway` until it has a manifest and code. |

This table is maintained by hand because the apps draw descriptions from mixed manifests (Cargo, package.json, Package.swift). When you add or rename an app, update this row and the app table in `CLAUDE.md`, then run `scripts/check-doc-drift.sh --refresh`.
